CsvPlotter

A simple, focused desktop tool for plotting and analysing time-series CSV data — a "Time" column plus N data columns. Built with Python + Qt Widgets (PySide6) and pyqtgraph. Current phase: MVP complete.

1. Project overview

CsvPlotter is a desktop application for plotting and analysing CSV data. The expected input is a CSV with one Time column (the X axis) and any number of numeric data columns. The user opens the file, picks which columns to show, and explores the signals interactively: zoom, pan, a hover tooltip, per-series scale (gain) and offset, and a pair of pinned points for reading the X/Y difference between two points.

The whole thing is Python + Qt Widgets (PySide6) with pyqtgraph doing the plotting and pandas / numpy doing the CSV loading and the maths. It is deliberately kept small and focused — an everyday "open a log and look at it" tool, not a replacement for a full analysis suite.

It is a sibling to AutoLabel and InferenceVisualizer and follows the same documentation conventions (this index.html + a shared style.css, a phased roadmap, and a changelog). Those two are C++/Qt; CsvPlotter is the Python/Qt member of the family, because the Python plotting ecosystem (pyqtgraph) gives interactive scientific plotting almost for free.

2. Current phase MVP complete

The design and scope below are agreed but not yet implemented. The MVP is the user's core feature list plus two-point pinned measurement:

  • Core: open a CSV (Time + N columns), select which columns to plot, zoom, pan, a hover tooltip, per-series scale (multiply) and X/Y offset.
  • Agreed extra: click-to-pin points (max two) with a Δx / Δy readout.
  • Deferred to roadmap: reset/auto-fit + PNG export, automatic downsampling for large files, session save/load, and a visible-range statistics panel.

Packaging (later)PyInstaller for a standalone Windows build

Why pyqtgraph and not Matplotlib? pyqtgraph is built for interactive data exploration: mouse-wheel zoom, box-zoom, and crosshair primitives are part of its ViewBox out of the box, and it stays fast on large arrays. Matplotlib excels at static, publication-quality figures but its interaction is slower and clunkier — the wrong trade for a "load a log and scrub through it" tool.

5. How it works (architecture)

The design follows the same separation the other two projects use: a thin UI layer, a data model, and a MainWindow that wires them together. Signals flow one way: the user changes something in the side panel → the model updates → the plot re-renders.

The Series model

Each data column becomes one Series object holding the raw, untouched arrays plus its display parameters: 

@dataclass
class Series:
    name: str            # column header
    x: np.ndarray        # raw Time values
    y: np.ndarray        # raw data values
    visible: bool = True
    gain: float = 1.0    # multiply Y by this
    offset_x: float = 0.0
    offset_y: float = 0.0
    color: str = ...     # auto-assigned from a palette

    def plot_x(self): return self.x + self.offset_x
    def plot_y(self): return self.y * self.gain + self.offset_y

The key idea: the raw data is never mutated. Scale and offset are display parameters, and the plotted arrays are computed on the fly (plot_x / plot_y). That keeps "reset to original" trivial and makes the transforms cheap to tweak live.

The render loop

  1. csv_loader reads the file with pandas, identifies the Time column, and builds one Series per remaining numeric column.
  2. MainWindow hands the series to the SeriesPanel (which lists them with checkboxes + gain/offset spin-boxes) and to the PlotView.
  3. Any change in the panel — toggle visibility, edit gain/offset — emits a signal; MainWindow tells PlotView to redraw the affected curve from its plot_x() / plot_y().
  4. Zoom/box-zoom, the hover tooltip, and click-to-pin are handled inside PlotView by a custom PlotViewBox plus a SignalProxy (mouse-move) and sigMouseClicked.

6. Feature design

Open CSV & column selection

File → Open CSV… (Ctrl+O). The loader looks for a column literally named Time; if it isn't found, it falls back to the first column and the user can re-pick the Time column from a dropdown. Every other numeric column appears in the SeriesPanel as a checkable item — tick to plot, untick to hide.

Mouse: zoom & box-zoom

A custom PlotViewBox (subclass of pyqtgraph's ViewBox) sets the mouse policy: left-drag draws a rubber-band box and the view zooms to it on release (RectMode); middle-drag pans; the mouse wheel zooms; the right button is inert (context menu disabled, no right-drag scaling). A left-click near a signal is reserved for pinning a point (below), so the left button no longer pans. View → Reset View (Ctrl+R) re-frames all visible data via autoRange().

Step lines (sample & hold)

By default each series is drawn as a stair: a sample's value holds until the next sample's time, which is how embedded-system / digital logs actually behave (the value doesn't ramp linearly between log entries). This uses pyqtgraph's stepMode="right". View → Step Lines toggles it off for ordinary linear interpolation when a series is genuinely continuous.

Hover tooltip

The tooltip only appears when the cursor is near a signal (within ~20 px). When it is, a marker snaps to the nearest sample of the closest curve and a small box shows that point's X and Y. Proximity is measured in pixels (the candidate sample per series is found with np.searchsorted on the plotted x, then mapped to screen coordinates), so "close enough" is consistent regardless of axis scale. Wired through a pyqtgraph SignalProxy on mouse-move so it's rate-limited and stays smooth; it hides in empty space and off the plot. No in-plot legend — the side panel already shows which colour is which.

Scale (gain) and offset

Each series row in the panel carries three spin-boxes: Gain (Y is multiplied), Offset Y, and Offset X (shifts the curve along time). These map directly to the Series fields, and editing one redraws just that curve. Useful for overlaying signals of very different magnitudes, or aligning two traces in time.

Pinned points (Δx / Δy)

Left-clicking near a signal pins a point there (a square marker snapped to the nearest sample, with an X/Y label). At most two pins exist at once — clicking a third drops the oldest (FIFO queue). The bottom of the left tab lists where the pins are (signal name + X/Y of each) and the Δx / Δy between them, with a Clear points button. This is the main "analysis" feature — measuring intervals and value changes directly off the plot, between any two points on any signals.

7. Build & run instructions

No compiler needed — it's pure Python.

# 1. Create and activate a virtual environment (Windows PowerShell)
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run
python main.py

requirements.txt:

PySide6
pyqtgraph
pandas
numpy

Later, a one-file Windows executable can be produced with pyinstaller --noconsole --onefile main.py.

Possible future implementations

  • Polish. Error dialogs for more edge cases, QSettings to remember window/dock layout, additional keyboard shortcuts.
  • PNG export of the current view.
  • Automatic downsampling so very large CSVs stay responsive.
  • Session save/load — persist column selection, gains, and offsets to a file.
  • Visible-range statistics — min / max / mean of what's on screen.
  • Multi-file overlay — plot more than one CSV at once.
  • Time-column override — a dropdown to re-pick which column is the X axis.
